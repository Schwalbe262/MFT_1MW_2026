1. `Y:\git\MFT_1MW_2026\HANDOVER.md`를 정독해라 — 미션, 스펙, 시스템 구조,
   피눈물 교훈 9개조, 다음 할 일이 전부 거기 있다.
2. 메모리 `C:\Users\peets\.claude\projects\Y--git-MFT-1MW-2026\memory\`의
   mft-campaign-directive, mft-infra-lessons를 읽어라.
3. 현재 가동 상태를 점검해라: train.parquet 행수, 스케줄러(127.0.0.1:8000)의
   활성 태스크, 로컬 생산 프로세스(python, run_simulation_260706 검색),
   백그라운드 피더/회수 루프 생존 여부. 죽어 있으면
   `regression_260707/campaign/relaunch.sh` 참고해 재가동.

**목표**: 검증 게이트 5단계를 통과한 최종 설계. 스펙 요지 — Llt 27.5µH ±2%(실물),
전 부품 ≤100°C, 부피+손실 최소화, 절연 40mm, B≤1.2T, 1kHz/1MW.
데이터 1만행+ → 서러게이트 → NSGA-2 → 능동학습 → fine 최종검증 순.

**운영 원칙 (어기면 하루를 날린다):**
- 코드 수정은 로컬 랜덤 3연속 완주 검증 후에만 클러스터 배포 (스모크만으로 금지)
- 함대 개입은 무손실로 (target 조정·큐 재활용). 전체 드레인은 치명 결함일 때만
- 동시수는 코어 예산(free_cpus/4 × 0.85)으로, 처리량 지표는 train.parquet 행수/시간만 신뢰
- 리눅스 gRPC 추출 API는 불신 (field summary/get_solution_data가 신뢰 경로)
- TaskStop 후엔 powershell로 파이썬 자식 생존 확인

**보고 스타일**: 사용자는 한국어, 정직한 수치 우선, 문제는 원인·증거·처방 세트로.
막히면 추측 말고 로그를 읽어라. 사용자가 GUI로 직접 재현해주는 정보가 가장 값지다.
