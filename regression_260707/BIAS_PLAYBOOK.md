# 서러게이트 편향 대응 플레이북

NSGA-2가 내놓은 "최적" 설계를 FEA로 검증했을 때 예측과 어긋나는 상황별 진단·대응 절차.
전제: 검증 라운드마다 `al_rounds/round_XX/verification_errors.csv`에 후보별
(예측, 실측, 오차)가 기록된다. **판정은 항상 이 실측-예측 잔차 분포로 한다.**

## 0. 편향의 4가지 얼굴 (잔차 분포로 구분)

| 패턴 | 증상 | 주원인 | 1차 대응 |
|---|---|---|---|
| A. 낙관 편향 (한쪽으로 치우침) | 검증 후보 대부분이 예측보다 나쁨 (Llt 이탈/온도 초과) | **옵티마이저의 오차 착취** — 모델이 낙관적으로 틀리는 영역으로 수렴 | q 조임 강화 (자동: 예측통과/실측탈락 시 q×1.25), 밀도 게이트 분위 95→90% |
| B. 무작위 산포 (양쪽 대칭) | 오차 부호가 반반, 크기만 큼 | 데이터 부족 / 라벨 노이즈 | 인필 계속 (라운드당 33개가 최적점 주변을 채움), 라벨 노이즈면 pe 조임 |
| C. 계통 오프셋 (전 후보 동일 방향·비슷한 크기) | 예: 모든 후보 Llt가 일관되게 +3% | **기준 불일치/물리 결함** — ×2 환산 누락, 단위, 보정계수, 코드 버전 혼합 | 파이프라인 감사 (아래 체크리스트) — 데이터를 더 모아도 해결 안 됨 |
| D. 특정 영역만 실패 | 특정 파라미터 구석의 후보만 어긋남 | 커버리지 공백 / 그 영역의 물리 특이 (예: 포화, 접촉) | 해당 영역 부스터 샘플링, 필요 시 설계공간에서 절제 |

## 1. 라운드 종료 시 자동 판정 (al_driver CHECK 단계에 구현됨)

- 예측통과 후보가 실측 탈락 → **q_mult ×1.25** (최대 3×q90): 다음 라운드에서 제약이 조여짐
- 전원 통과 + 여유 2배 이상 → q_mult ×0.9 (과보수 완화)
- 종료: 실측 전 스펙 통과 ≥3 + 배치 일치도 (|ΔLlt| 중앙값 ≤0.5%/최대 ≤1%, |ΔT| ≤3/5°C)

## 2. 계통 오프셋(C) 감사 체크리스트 — 순서대로

1. **git_hash 혼합**: 잔차를 git_hash별로 그룹핑 — 버전 간 평균이 다르면 해당 해시 데이터 격리
   (실사례: 프로브 전치 수정 전 행들의 side/core 온도 → 회수기에서 자동 NaN 처리 중)
2. **기준 환산**: Llt는 대칭 매트릭스 ×2 = 실물 — 학습 타겟(Llt_phys)과 제약(27.5)이 같은 기준인지
   (`to_physical`과 NSGA-2 제약이 유일한 환산 지점이어야 함)
3. **보정계수**: 대칭 loss의 2^c/4, 2^c/2^1.74 — 게이트1 대조(±1%)를 최신 코드로 재확인
4. **여자/운전점**: 검증 프로파일(P_target, pe)이 캠페인과 동일한지 (`profiles/standard.json`)
5. **단위**: 새 컬럼 추가 시 [unit] 접미사 확인 (`_unit_scale`) — mA 사건 재발 방지
6. 원인 확정 후: 오염 데이터는 **삭제하지 말고 격리 컬럼 NaN 처리** (유효 컬럼은 재활용)

## 3. 낙관 편향(A)이 q 조임으로도 안 잡힐 때 (에스컬레이션)

1. 밀도 게이트 분위 95% → 90% → 85% 단계 강화 (외삽 봉쇄 반경 확대)
2. 탐사 후보 비율 확대 (K=33 중 탐사 10 → 16): 모델이 모르는 곳을 우선 실측
3. 검증 실패 후보 자체를 학습에 투입 (sample_weight 3 → 5): "그 지역이 나쁘다"는 정보 주입
4. 최후: 옵티마이저 목표를 완화하고 (예: 부피 하한 제약 추가) 인간 검토 요청

## 4. 온도 특이사항

- 온도 편향은 손실 편향의 **하류**다: ΔT가 크면 먼저 P_winding/P_core 잔차를 봐라
  (손실이 맞는데 온도만 틀리면 열모델(부력/메시/균질화) 쪽, 손실도 틀리면 EM 쪽)
- 1/8 열모델은 부력 무시로 핫스팟 −2~3°C 계통 (게이트1 실측) → quarter 전환으로 제거됨.
  eighth 데이터가 섞여 있으면 thermal_symmetry 컬럼으로 구분 가능

## 5. 리허설에서 확인된 것 (2026-07-07, 16행)

- 16행 학습 → NSGA-2 → 후보 선정 → 검증 제출까지 기계적으로 완주 (지표는 무의미: Llt MAPE 39%)
- 이 상태의 모델로 NSGA-2를 돌리면 전형적 A형(착취) 결과가 나옴 — q90이 이미 2.5~10으로
  커서 컨포멀 조임이 제약을 사실상 봉쇄 → "데이터 부족 시 스스로 보수화"가 의도대로 작동
